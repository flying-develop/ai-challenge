# Инварианты переходов задачи

## Обязательные

- Переход planning → execution ТОЛЬКО при наличии утверждённого артефакта плана (ARTIFACT:PLAN)
- Переход execution → validation ТОЛЬКО при наличии артефакта результата (ARTIFACT:RESULT)
- Переход validation → done ТОЛЬКО при наличии артефакта валидации (ARTIFACT:VALIDATION)
- Прямой переход planning → validation ЗАПРЕЩЁН: нельзя валидировать без выполнения
- Прямой переход planning → done ЗАПРЕЩЁН: нельзя завершить без выполнения и валидации
- Прямой переход execution → done ЗАПРЕЩЁН: нельзя завершить без валидации
- Обратный переход в предыдущее состояние ЗАПРЕЩЁН (нет execution → planning)
- Пауза (pause) доступна из любого активного состояния (planning, execution, validation)
- Resume восстанавливает ТОЛЬКО то состояние, из которого была пауза
- При resume артефакты предыдущих этапов ОБЯЗАНЫ присутствовать в контексте LLM

## Рекомендуемые

- В planning желательно минимум 2 turn-а диалога перед формированием плана
- В execution желательно подтверждение каждого шага перед переходом к следующему
- В validation желательно проверять каждый пункт плана отдельно
